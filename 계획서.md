# 📋 계획서 — Auto-Pilot 자율 루프

Auto_Pilot.py + MCP filewriter(write_file + signal_pause)를 결합해, Claude 챗 앱이 혼자 계획서를 다듬고 → .md로 저장하고 → 안정되면 스스로 멈추는 완전 자율 루프를 구동한다.

## 다음 한 걸음
Claude 챗 앱 **완전 재시작**(트레이까지 종료) → 채팅에 "사용 가능한 도구 알려줘" → `write_file`, `signal_pause` 보이면 MCP 연결 성공.

## 검증 체크리스트
**단계 1 — MCP 연결 확인 (챗 앱 재시작 후)**
- [ ] 채팅에 "사용 가능한 도구 알려줘" → `write_file`, `signal_pause` 두 개 표시
- [ ] "signal_pause 호출해" → `AutoPilot\PAUSE.flag` 파일 생성 확인
- [ ] "filename=테스트.md, content=hello 로 write_file 호출해" → `AutoPilot\테스트.md` 생성 확인

**단계 2 — 자율 루프 통합 테스트**
- [ ] Auto_Pilot.exe 실행 → 연속 모드 ON
- [ ] 몇 주기 후 `AutoPilot\계획서.md` 내용이 갱신되는지 확인
- [ ] 계획 안정 시 연속 모드 체크박스가 자동으로 꺼지는지 확인

## 결정 로그
- Ctrl+A/C 클립보드 감지 → **PAUSE.flag 파일 방식**으로 전환 (클립보드 충돌 해소)
- `uvx --from mcp mcp run` → **`uv run --with mcp python`** (typer 미포함 이슈 해소, 실제 검증)
- 일시정지 게이트를 GENERATING→done 분기에서 **RESUMING 분기 단일 위치**로 이동 (모든 경로 커버)
- `write_file` 경로 제한: AUTOPILOT_DIR 하위 파일명만 허용 (경로 탈출 방어)

## 열린 질문
- Store 버전 Claude 챗 앱에서 MCP 서버가 실제로 뜨는지 미검증
- `uv run --with mcp` 첫 실행 cold-start 지연 (패키지 다운로드 수 초) → 타임아웃 이슈 위험
  - **완화 방안**: Claude 챗 앱 재시작 전 터미널에서 `uv run --with mcp python mcp_filewriter.py` 한 번 수동 실행해 패키지를 캐시해두면 이후 즉시 기동됨
