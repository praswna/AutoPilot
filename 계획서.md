# 📋 계획서 — Auto-Pilot v2.0

## 목표
다중 스텝 입력 UI + 완료 토큰 감지(템플릿·OCR) + 사전 검증으로 구성된 스텝 기반 자율 루프.
클래식 모드(SMART_PROMPT)와 공존하며, 텔레그램 양방향 제어·F12 긴급 정지를 갖춘다.

## v2.0 아키텍처 요약

### 상태 머신
```
IDLE → MONITORING ←→ GENERATING
          ↓               ↓ (done)
       WAITING        HANDLE_DONE
          ↓               ├─ 스텝 완료? → expected_step++ → RESUMING
       RESUMING        ├─ 계속 필요? → continue_count++ → RESUMING / PAUSED
          ↑            └─ 클래식: LOOPSTABLE? → MONITORING
          └───────────────────────────────────────────────────┘
```

### 스텝 완료 감지 우선순위
1. `step{N}_complete*.png` 존재 시 → 템플릿 매칭
2. 없으면 → OCR으로 `[STEP{N}_DONE]` 토큰 검사
3. 둘 다 불가 → 사전 검증 단계에서 차단

### 텔레그램 양방향
- `TelegramPollerThread`: `getUpdates` 5초마다 폴링
- 수신자 필터: config.json의 `chat_id`와 일치하는 메시지만 처리
- `/next` → `force_next_step()` → PAUSED 탈출
